# 18 Chữ — Vietnamese word game

**Date:** 2026-07-10
**Route:** `/hogwarts/games/18-chu`
**Names:** VI "18 Chữ" · EN "18 Words"
**Inspiration:** [18words.com](https://18words.com/) — mechanics only. All code and word data are original / open-licensed. Nothing is copied from the source game.

## Goal

Replicate the *feel* of 18words for Vietnamese: 18 scrambled words solved in sequence
against a per-word countdown, one mistake ends the run, score is how far you got.

## Core mechanic

- 18 two-syllable Vietnamese words per run, shown one at a time.
- Each word is a scrambled row of letter tiles **with diacritics intact**:
  `Đ H I N A G Ì` → `GIA ĐÌNH`. Diacritics are strong constraints; the real
  puzzle is finding the syllable split, which is *never shown* while solving.
- **Timer counts down from 30s per word and resets to 30 on each new word**
  (the original's deliberately calmer choice, not an ascending clock).
- Solve a word → advance, timer resets. One wrong guess **or** one timeout →
  run ends immediately. Score = words survived, 0–18.
- **Difficulty curve by slot:** slot _i_ draws from its own length+frequency
  bucket. Word 1 is short (5 letters) and high-frequency; word 18 is long
  (up to 9) and rare — **and drawn from Lộc's own published Pensieve posts.**

## Input

- **Click/tap is primary** (no VN IME required). Click a tray tile → fills the
  next empty slot. Click a filled slot → returns its tile to the tray.
- Buttons: **Shuffle** (rearrange tray), **Backspace** (undo last placement),
  **Submit** (Enter). Reveal-letter hint ×3.
- Keyboard: typing a character that matches a tray tile exactly places it. Works
  only with a VN IME; never required.
- Syllable split renders as a gap **only after** the word is solved.

## Why there is a small server surface (the spoiler problem)

The daily is seeded by date. If the puzzle generator ran in the browser, its
bundle would *be* the answer key — a Tuesday player could read Wednesday's
words. So the word bank lives **server-side only** and one endpoint owns it.
Answers are **never** sent as plaintext.

### Endpoints (Astro API routes, `prerender = false`, read `env.DB`)

1. `GET /api/games/18-chu/puzzle?day=YYYY-MM-DD&mode=daily|practice|relax`
   - Seeds a PRNG (daily: from `day`; practice: from a random seed echoed back).
   - Selects 18 words from per-slot length+frequency buckets; word 18 from the
     Pensieve-corpus pool.
   - Returns per word: `{ tiles: string[], length: number, splits: number[], hash: string }`
     where `hash = sha256(dailySalt + normalize(answer))` and
     `normalize` = NFC, lowercased, syllables concatenated **without** space.
   - `dailySalt` is derived from `day` and shipped (it only blocks cross-day
     rainbow tables). **No plaintext answer, ever.**
   - `splits` = tile offset(s) of syllable boundaries, used only to render the
     gap *after* solving. Leaking "space after 3rd tile" does not spoil a
     7-tile anagram.
2. `GET /api/games/18-chu/hint?day&mode&seed&word=i&pos=j` → one letter.
   Capped at 3 per run (client-enforced, trusted).
3. `POST /api/games/18-chu/result { day, score, timeMs }` → insert into D1.
   `GET /api/games/18-chu/result?day=…` → your percentile among **scores
   submitted today**, plus play count. Trusted; **labelled honestly** as
   "Top X% of scores submitted today" (never "of players").

### Verification is client-side

Client hashes the assembled guess with `dailySalt` and compares to the word's
`hash`. So after the initial puzzle fetch, play is **offline-capable** — no
per-word round-trips, no dropped-connection deaths mid-run.

**Accepted residual risk:** a determined person could brute-force the 18 hashes
against their own Vietnamese word list. Nobody does that by accident at 2am.
Scores are trusted; the honest label keeps the percentile from being a lie.

## Local-only state (`localStorage`)

- Current streak, longest streak.
- Per-day completion map → drives the **archive** grid.
These are personal (only Lộc sees his own), so client trust is correct here.

## Modes

- **Daily** — timed, one puzzle/day shared by all, feeds the leaderboard.
- **Practice** — random seed (echoed back so it's shareable), timed, no submit.
- **Relax** — untimed, no submit.
- **Archive** — the daily endpoint with a past `?day=`.

## Word bank (build-time, server-only)

`scripts/build-18chu-bank.ts`:

1. Pull two-syllable compounds from open CC BY-SA sources
   ([duyet/vietnamese-wordlist](https://github.com/duyet/vietnamese-wordlist),
   [minhqnd/dictionary](https://github.com/minhqnd/dictionary)).
2. Keep entries that are: exactly 2 syllables, 5–9 total letters, valid
   Vietnamese syllable structure, no proper nouns.
3. Rank by corpus frequency; keep top ~3,000.
4. Bucket into 18 slots by length+rarity for the difficulty curve.
5. Second pass: extract 2-syllable words from **published** Pensieve posts →
   word-18 pool (intersected with the valid-compound set).

Output `data/18chu-bank.json` — imported by the puzzle route, **never placed in
`public/`**. CC BY-SA attribution line rendered in the game footer.

### Bank JSON shape

```jsonc
{
  "attribution": "…CC BY-SA sources…",
  "slots": [ /* 18 arrays; slots[i] = candidate words for slot i */
    ["nhà cửa", "bạn bè", …],
    …
  ],
  "corpusPool": ["…words from Lộc's posts…"]  // word 18 drawn from here
}
```

Answer strings stay server-side; only tiles + hashes cross the wire.

## D1 schema

`scripts/18chu-001-schema.sql` (run with `wrangler d1 execute pensieve-db --remote`):

```sql
CREATE TABLE IF NOT EXISTS game_18chu_results (
  id         TEXT PRIMARY KEY,       -- ULID
  day        TEXT NOT NULL,          -- YYYY-MM-DD
  score      INTEGER NOT NULL,       -- 0..18
  time_ms    INTEGER NOT NULL,
  created_at TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_18chu_day_score ON game_18chu_results(day, score);
```

Percentile = fraction of today's rows with `score <= yours`.

## Verification

- `SELECT COUNT(*) FROM game_18chu_results WHERE day='<today>'` returns rows after a run.
- `curl …/puzzle?day=<today>` returns 18 words, **no plaintext answer field**.
- Assembling the correct tiles client-side advances; wrong / timeout ends the run.
- Bank build: `node scripts/build-18chu-bank.ts` writes ~3,000 words, all 2-syllable, 5–9 letters.
