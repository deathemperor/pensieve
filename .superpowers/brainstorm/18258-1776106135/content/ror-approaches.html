Room of Requirement: Restructure Approaches

The page has 7 sections and growing. Which strategy handles scale best?

A

Hub + Sub-pages

RoR index becomes a compact dashboard with summary cards linking to dedicated sub-pages. Each growing section (Task Observatory, Restricted Section, House-Elves) gets its own page. Index stays short — just hero + summary grid.

Pros

  • Index never grows — new sections = new sub-pages
  • Each page loads only what it needs
  • Already have the pattern (Task Observatory)

Cons

  • Loses the "one glance" overview feel
  • More navigation clicks
B

Sticky Section Nav

Keep all sections on one page but add a sticky horizontal nav bar below the hero. Anchor links jump to sections. Scrollspy highlights the current section. Page stays long but navigable.

Pros

  • Everything on one page — no context switching
  • Fast jump to any section
  • Common pattern (Linear, Vercel docs)

Cons

  • Page keeps getting longer
  • Needs JS for scrollspy (or CSS-only with scroll-margin)
  • Mobile nav gets crowded with 7+ items
C

Hybrid: Hub Dashboard + Expandable Panels

RoR index is a dense grid of summary panels. Each panel shows key stats inline (like a dashboard). Clicking a panel either expands it in-place or navigates to its sub-page for growing sections. Static sections (hero, roles, source) stay collapsed/inline.

Pros

  • Best of both — overview at a glance, depth on demand
  • Growing sections get sub-pages; static sections stay inline
  • Scales infinitely

Cons

  • More design work upfront
  • Need to decide which sections expand vs navigate
