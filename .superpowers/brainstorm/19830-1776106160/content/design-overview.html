Design: Magical Creatures Rewrite

Filled silhouettes + upgraded CSS animations — same 6 creatures

SVG Changes

Before (stroke-only)
Open lines, not recognizable at small size
After (filled silhouette)
Solid shape, reads as "deer" instantly

Animation Upgrades

Easing

Replace ease-in-out with cubic-bezier(.23,1,.32,1) (ease-out-quint) for traversal. Faster start, gentle deceleration — feels alive, not mechanical.

Enter / Exit

Fade in over 300ms (ease-out), fade out over 200ms (ease-in). Currently both are baked into the traversal keyframe at 5%/95% — split them for proper timing.

Motion

Ground: subtle body tilt ±2° synced with stride. Flying: wing-beat via scaleY pulse. Replace the jerky mc-gallop with smoother curves.

Glow

Pulsing drop-shadow on a slow cycle (2-3s) — the creature "breathes light". Currently static filter.

Accessibility

Wrap all animation in @media (prefers-reduced-motion: no-preference). With reduced motion: creature still appears but no traversal, just a soft fade in/out in place.

Performance

Only animate transform + opacity (GPU-composited). Move glow pulse to a separate layer with will-change: filter. Zero layout thrash.

Behavior (unchanged)

  • Random creature every 45-90s
  • Random direction (left→right or right→left), random vertical position (20-80vh)
  • Only one creature at a time
  • Click footer name → summon stag + sparkle sound
  • window.MagicalCreatures API unchanged: start(), stop(), summon(name)

Scope

Single file rewrite: public/js/creatures.js. No new dependencies. No changes to Base.astro integration. Drop-in replacement.
