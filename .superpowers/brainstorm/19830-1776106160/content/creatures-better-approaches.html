Better Animal Rendering

Three approaches — click buttons to see each one live

Recommended Emoji Patronus

Use native emoji animals, blast them white with CSS brightness(100) saturate(0), add patronus glow via drop-shadow. The OS renders a detailed animal — we turn it into a ghost. Zero drawing, instantly recognizable.

Recognizable?
★★★ Instantly — OS-drawn art

~2KB JS (no SVG data)
Cross-platform
Looks different per OS — Apple/Google/Windows emoji differ
Patronus feel
Strong — glowing white silhouette from detailed base
Perf
GPU-composited filters + transforms only

Alternative External SVG Silhouette Files

Use proper SVG silhouette files stored in public/svg/creatures/. Source from open-source icon libraries (Lucide, Phosphor, or free silhouette sets). Professional artwork, consistent style, but requires finding/curating the right files.

Can't demo this live without the actual SVG files — the quality depends entirely on sourcing good silhouettes. Options: Phosphor Icons (has cat, dog, bird, horse), SVG Repo (free silhouettes), or custom illustrations. This is the "most work upfront, best long-term control" approach.

Alternative Lottie Animations

Use pre-made Lottie walk/fly cycle animations, tinted to patronus colors. The animals actually move — legs walk, wings flap with real keyframed motion. Highest visual ceiling, but adds lottie-web (~50KB) dependency.

Can't demo without Lottie files, but imagine: a stag that actually walks with leg motion, an owl with individually animated wing feathers. Sources: LottieFiles.com has free animal walk cycles. Tint to patronus blue-white via Lottie color manipulation.

Trade-off: +50KB for lottie-web, need to find/license 6 matching animations, more complex integration.
